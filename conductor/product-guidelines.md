# プロダクトガイドライン: FaveFit

## 1. ビジュアルアイデンティティ
- **第一印象:** 清潔感、フレッシュ、鮮やか。
- **カラーパレット:**
  - **メインカラー:** フレッシュグリーン（健康と野菜を象徴）＆ エネルギッシュオレンジ（食欲と活力を象徴）。
  - **背景色:** クリーンホワイト ＆ ライトグレー（読みやすさと清潔感を重視）。
  - **アクセント:** ウォームイエロー（ポジティブなフィードバックや達成感の表現に使用）。
- **デザインスタイル:** モダン ＆ クリーン（Tailwind CSS と shadcn/ui を活用）。
  - 認知負荷を下げるために余白を贅沢に使用する。
  - 食欲をそそる高品質な料理画像を使用する。

## 2. ボイス ＆ トーン（言葉遣いと雰囲気）
- **核心となる姿勢:** 「ポジティブな肯定（Positive Reinforcement）」。
- **ガイドライン:**
  - **常に肯定する:** たとえ計画通りにいかない時でも、次のポジティブな一歩にフォーカスする。決して叱ったり、罪悪感を持たせたりしない。
  - **励ましの言葉:** 「ナイスチョイス！」「美味しそうなアイデアですね！」「一緒に楽しみましょう！」といった前向きな表現。
  - **簡潔で明快:** 専門用語を避け、栄養学的なメリットも日常的な言葉で説明する。

## 3. ユーザー体験（UX）の原則
- **スピードこそ正義:** 「気分の選択」から「レシピの提案」までの流れは、ユーザーの直感を逃さないよう、極めてスムーズかつ迅速であること。
- **視覚的な分かりやすさ:** 栄養データは数字の羅列ではなく、グラフやシンプルなアイコンを用いて直感的に理解できるようにする。
- **感動の瞬間（Magic Moments）:** ユーザーが言語化できていなかった「まさに今これが食べたかった！」という欲求をAIが提案した時に、最大の価値が生まれる。

## 4. インタラクションパターン
- **入力:** タイピングを最小限に抑える。「気分」の選択にはタップ、スライダー、ビジュアルセレクターを活用する。
- **フィードバック:** レシピの感想入力は、アンケートではなく「AIとの気軽なハイタッチ」のような軽快な体験にする。
