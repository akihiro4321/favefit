# Track Specification: Core UI Prototype

## 1. 概要
本トラックでは、FaveFitのコアとなるユーザー体験（UX）を検証するためのUIプロトタイプを構築する。実際のバックエンド（Mastra/Gemini）との接続は行わず、モックデータを使用して、画面遷移とインタラクションの確認を目的とする。

## 2. 実装範囲
以下の3つの主要画面と、それらをつなぐナビゲーションを実装する。

### 2.1 ホーム（今日の気分入力）
- **機能:** ユーザーがその時の気分（ジャンル、味、スタイル）を直感的に選択する。
- **UI要素:**
  - タップ可能なジャンルカード（和食、中華、洋食...）
  - スライダーまたはボタンによる味の調整（さっぱり ↔ こってり）
  - フリーテキスト入力エリア
  - 「レシピを見る」ボタン

### 2.2 レシピ一覧（提案結果）
- **機能:** 選択した気分に基づいて生成された（モック）レシピを3案提示する。
- **UI要素:**
  - レシピカード（料理写真、タイトル、カロリー、調理時間）
  - 「詳細を見る」アクション

### 2.3 レシピ詳細
- **機能:** レシピの具体的な作り方と栄養情報を表示する。
- **UI要素:**
  - 材料リスト
  - 調理手順（ステップバイステップ表示）
  - 栄養素グラフ（PFCバランス）
  - 感想入力ボタン（モック）

## 3. 技術仕様
- **Framework:** Next.js 16 (App Router)
- **Styling:** Tailwind CSS
- **Components:** shadcn/ui
  - Card, Button, Slider, Badge, Dialog 等を使用
- **Icons:** Lucide React
- **Mock Data:** TypeScriptで型定義されたJSONデータを使用

## 4. デザイン要件
- **テーマカラー:** フレッシュグリーン (#4ADE80) & エネルギッシュオレンジ (#FB923C) を基調とする。
- **レスポンシブ:** モバイルファーストで実装し、PWAとしての操作感を重視する。
